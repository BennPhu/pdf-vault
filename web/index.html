About PDF Vault

    

      
      
      
    

    

      

        
    
          
  • Add files — drop PDFs or images (PNG/JPEG/WebP) onto the drop zone, or click to browse. Images are converted to PDFs automatically.
    • 
          
  • Search — type in the box atop the Library; press Esc to clear.
    • 
          
  • Rename — double-click a file's name in the Library.
    • 
          
  • Preview — click any PDF to flip through its pages on the right.
    • 
          
  • Merge / Split — select files, then use the bottom bar: merge into one, save a page range, or split every page to its own file.
    • 
          
  • Edit Pages — select one PDF to rotate, reorder, or delete pages.
    • 
          
  • Compress — shrink selected PDFs in place; originals are kept in trash so you can undo.
    • 
          
  • Delete & Undo — deleted files go to a hidden trash for 30 days; use Undo/Redo in the Library header.
    • 
          
  • Master PDF — bind your entire library into a single document.
    • 
          
  • Storage — everything lives in the folder you chose; change it anytime via ⚙ Storage.
    • 
        

      

      

        
macOS can already merge PDFs — that was never the point.

        
PDF Vault is about controlling and editing your PDFs on your own terms, without uploading them to anyone's cloud. Reorder, rotate and delete pages, split, compress, rename, and organize an entire library — edits that native tools scatter or bury, and that online PDF sites only offer in exchange for your documents.

        
It sits deliberately between the native PDF tools and the online ones: more capable and organized than Preview, and 100% private where the web tools are not. Your files never leave your machine — no accounts, no ads, no telemetry, no cloud. The only network request ever made is an optional, checksum-verified update check against GitHub Releases.

        
And because it is free and open source (MIT), you don't have to take our word for any of this — anyone can read the code and verify exactly what happens to their files.

        
A note on memory: the app idles around ~200 MB. That is the fixed cost of its rendering engine (the same WebKit that powers Safari) plus the Python runtime — your PDFs are never held in memory, and comparable desktop apps typically use 2–4× more.
